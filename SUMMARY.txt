📊 DATABASE SCHEMA:

┌─ auth.users (Supabase built-in)
│  ├─ id (UUID)
│  └─ email, password_hash, etc.
│
├─ products
│  ├─ id (TEXT) ← product ID
│  ├─ name, brand, category
│  ├─ price (Rupiah)
│  ├─ image (URL)
│  ├─ stock
│  └─ store_id (FK)
│
├─ stores
│  ├─ id (TEXT)
│  ├─ name, address, phone, hours
│  ├─ query (for maps)
│  ├─ social (JSON)
│  └─ is_visible (boolean)
│
├─ wishlists ⭐ NEW!
│  ├─ id (auto)
│  ├─ user_id (FK to auth.users) ← Which user
│  ├─ product_id (FK to products) ← Which product
│  └─ UNIQUE(user_id, product_id)
│
├─ reviews
│  ├─ id (auto)
│  ├─ store_id (FK)
│  ├─ author
│  ├─ rating (1-5)
│  ├─ text
│  └─ date
│
└─ ... (customers, orders, order_items, admins)

🔐 ROW LEVEL SECURITY (RLS):

Table              │ Select    │ Insert           │ Update           │ Delete
───────────────────┼───────────┼──────────────────┼──────────────────┼──────────────
products           │ All users │ Admin only       │ Admin only       │ Admin only
stores             │ All       │ Admin only       │ Admin only       │ Admin only
(visible=true)     │           │                  │                  │
wishlists          │ Own user  │ Own user         │ N/A              │ Own user
reviews            │ All users │ Admin only       │ Admin only       │ Admin only
customers          │ Own user  │ Own on signup    │ Own user         │ Admin only
carts              │ Own user  │ Own user         │ Own user         │ Own user
orders             │ Own/admin │ Own user         │ Admin only       │ Admin only
order_items        │ Own/admin │ Admin only       │ Admin only       │ Admin only

🔑 KEY FEATURES EXPLAINED:

1. WISHLIST PERSISTENCE
   ├─ Data stored in DB, not local storage
   ├─ Tied to user account (via user_id)
   ├─ Unique constraint: 1 product per user max
   ├─ RLS policy: Only own user can see/modify
   └─ Auto-deletes from UI on logout (but stays in DB)

2. AUTHENTICATION
   ├─ Email + password signup
   ├─ Email confirmation (optional, configurable)
   ├─ Session managed by Supabase Auth
   ├─ currentUser set after login
   ├─ Auto-logout on signOut()
   └─ Credentials stored securely in auth.users table

3. PRODUCT MANAGEMENT
   ├─ All products stored in products table
   ├─ Linked to stores (store_id FK)
   ├─ Filterable by category, brand, price
   ├─ Searchable by name or brand
   └─ Editable via Supabase dashboard

4. REVIEWS & RATINGS
   ├─ Per-store reviews (linked via store_id)
   ├─ 1-5 star rating system
   ├─ Author name optional (default "Anonim")
   ├─ Date auto-set to today
   ├─ Displayed in product modal
   └─ Admin can delete reviews

5. PETA/MAPS
   ├─ Removed from main catalog
   ├─ Available in product modal (reviews section)
   ├─ Query string stored in stores.query
   ├─ Can use Google Maps API or Leaflet
   └─ (Implementation left for you)
